# Submit Code Skill

This skill helps you submit code changes to git with intelligent commit messages based on the changed files.

## When to Use
- When you want to submit code changes
- When you need to push changes to remote git
- When you want to ensure code quality before submission

## How It Works
The skill runs the following steps:

1. **Update AI log** - Updates AI_LOG.md with current task information
2. **Format code** - Runs `pnpm run format` to ensure code consistency
3. **Security check** - Invokes security-check skill to scan for vulnerabilities
4. **Run tests** - Runs `pnpm run test` to ensure code quality
5. **Build project** - Runs `pnpm run build` to verify build success
6. **Analyze changes** - Detects changed files and generates appropriate commit message
7. **Commit changes** - Commits changes with intelligent commit message
8. **Push to remote** - Pushes changes to remote git repository

## Analyze Changes Process

The analyze changes step is performed by the AI assistant to generate precise commit messages:

1. **Collect changed files** - The script shows all modified files via `git status --porcelain`
2. **Show diff summary** - Display `git diff --stat` to understand the scope of changes
3. **AI generates commit message** - Based on:
   - The current task context
   - The changed files and their purpose
   - Best practices for conventional commits

### AI Commit Message Generation

The AI assistant will analyze:
- What files were changed
- The purpose/impact of the changes
- Current task context from the conversation

And generate a short, precise commit message following these guidelines:
- Use conventional commits format (feat:, fix:, chore:, etc.)
- Keep under 72 characters
- Focus on "what" not "how"
- Be specific to the actual changes

Example commit messages the AI might generate:
- `feat: add block coverage tests for write page editor`
- `fix: resolve lodash import issue in excalidraw plugin`
- `chore: update deploy script with new steps`
- `docs: clarify code submission workflow`

## Usage

### Run the submission process
```bash
#!/bin/bash
set -e

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*"
}

log "=== Step 1: Updating AI log ==="
# Update AI_LOG.md with current task information
# This step is performed by the AI assistant based on the conversation context

log "=== Step 2: Running format ==="
pnpm run format

log "=== Step 3: Running security check ==="
# Invoke security-check skill to scan for vulnerabilities
# This will check for password leaks, API key leaks, and other security issues
npx trae run security-check

log "=== Step 4: Running tests ==="
pnpm run test

log "=== Step 5: Building project ==="
pnpm run build

log "=== Step 6: Analyzing changes ==="

# Get changed files with status
CHANGED_FILES=$(git status --porcelain)

if [ -z "$CHANGED_FILES" ]; then
    log "No changes to commit"
    exit 0
fi

log "Changed files:"
echo "$CHANGED_FILES"

# Get git diff to understand changes
git diff --stat

log "\n=== Step 7: AI generates commit message ==="

log "Changed files and diff summary displayed above."
log "AI assistant will analyze this information and generate a precise commit message."

# Note: The actual commit message will be generated by AI based on:
# - Current task context
# - Changed files analysis
# - Conventional commits best practices

log "=== Step 8: Committing changes ==="
git add -A
git commit -m "chore: update changes"  # AI will replace this with precise message

log "=== Step 9: Pushing to remote ==="
git push

log "=== Code submission completed successfully ==="
```

## Troubleshooting

### No changes to commit
If you see "No changes to commit", it means there are no files modified or staged.

### Security check failures
If security check fails, the process will stop. Review the security issues reported and fix them before running the skill again.

### Test failures
If tests fail, the process will stop. Fix the test issues and run the skill again.

### Build errors
If build fails, the process will stop. Fix the build issues and run the skill again.

### Git push issues
If git push fails, check your git credentials and network connection.
